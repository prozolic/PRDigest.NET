#### 概要
JIT コンパイラの式の畳み込み処理をリファクタリングし、二重 NOT/NEG 演算の最適化を `fgMorphSmpOp` から `gtFoldExpr` に移動しました。これにより不要なノードの早期削除が可能になり、測定可能なスループット改善が得られます。`gtFoldExpr` を単項/二項ノード向けのヘルパー関数に分割し、最適化ロジックを一元化しています。

#### 変更内容
- **src/coreclr/jit/morph.cpp**: `fgMorphSmpOp` から二重 NOT/NEG 削除ロジック（10行）を削除
- **src/coreclr/jit/gentree.cpp**: `gtFoldExpr` をリファクタリングし、ノードタイプに基づいてディスパッチするよう改善。`gtFoldExprUnary`/`gtFoldExprBinary` ヘルパーを追加（+115/-47行）し、二重 NOT/NEG 畳み込みを `gtFoldExprUnary` で実装
- **src/coreclr/jit/compiler.h**: 新しいヘルパー関数 `gtFoldExprUnary`/`gtFoldExprBinary` の宣言を追加（+2行）

#### パフォーマンスへの影響
スループット(TP)の測定可能な改善を達成。SPMI ベンチマーク結果（T0 と FullOpts 間）で効果を確認。改善の主要因は `gtFoldExpr` のチェック順序の再編成と `gtFoldExprUnary` の分割による早期最適化です。

#### 関連Issue
[#128187](https://github.com/dotnet/runtime/issues/128187)

#### その他
変更は JIT コンパイラの内部実装（gentree.cpp/morph.cpp）に限定され、公開 API には影響なし。Valnum-CSE 処理中の呼び出しに関する想定を厳密化しています。

#### 概要
JITコンパイラの値番号（Value Numbering）管理において、`O2K_CHECKED_BOUND_ADD_CNS`と`O2K_VN_ADD_CNS`の2つの演算子を統合しました。`O2K_VN_ADD_CNS`は「X+CNS」の形式を、Xの値番号とCNS定数に分解して表現するようにし、アサーション伝播時のホットループ処理を効率化しています。

#### 変更内容
- **compiler.h**: 値番号演算子の定義と関連インターフェースを統合・簡潔化（+30/-44行）
- **assertionprop.cpp**: アサーション伝播ロジックを統合版の新しい意味論に対応させ、`GetVNFunc`呼び出しの削減（+15/-27行）
- **rangecheck.cpp**: 範囲チェック処理を新しい分解された値番号体系に対応（+23/-18行）

#### パフォーマンスへの影響
**改善点**: MergeEdgeAssertions処理のホットループで`GetVNFunc`呼び出しを削減し、X（全体のADD演算ではなく左辺オペランド）の値番号に直接アクセスできるため、メモリアクセスとの呼び出しオーバーヘッドが削減される可能性があります。

#### 関連Issue
なし

#### その他
- 非負性保証の扱いを明確化：`O2K_VN_ADD_CNS`単体では非負性を暗に示さず、必要に応じて`GetOp2().IsVNNeverNegative()`で明示的に確認するように修正
- 境界チェックノードが2つのアサーションを生成できるよう将来的に改善する検討中（現状はツリーあたり最大1つのアサーションのため制限あり）

### [#122607](https://github.com/dotnet/runtime/pull/122607) \[browser\] Library mode without workload {#122607}
- 作成者: [@maraf](https://github.com/maraf)
- 作成日時: 2025年12月17日 13:36:10(UTC)
- マージ日時: 2026年05月16日 18:42:25(UTC)
- ラベル: <a style="text-decoration:none;" href="../../labels/arch-wasm/index.html"><span style="background-color: #eb6420; color: #ffffff; display: inline-block; padding: 0 7px; font-size:12px; font-weight:500; line-height:18px; border-radius:2em; border:1px solid transparent;">arch-wasm</span></a> <a style="text-decoration:none;" href="../../labels/area-Build-mono/index.html"><span style="background-color: #d4c5f9; color: #000000; display: inline-block; padding: 0 7px; font-size:12px; font-weight:500; line-height:18px; border-radius:2em; border:1px solid transparent;">area-Build-mono</span></a> <a style="text-decoration:none;" href="../../labels/os-browser/index.html"><span style="background-color: #FEF2C0; color: #000000; display: inline-block; padding: 0 7px; font-size:12px; font-weight:500; line-height:18px; border-radius:2em; border:1px solid transparent;">os-browser</span></a>
#### 概要
WebAssembly ライブラリモード（`RID=browser-wasm`、`OutputType=Library`）のビルドで、WASM ワークロードのインストールを不要にする変更。ライブラリプロジェクトは `SelfContained` や `PublishTrimmed` をデフォルトで有効にすべきではないという設計方針に基づいている。

#### 変更内容
- **ILLink ターゲット移行**: `_RootEntireIntermediateAssembly` を `BrowserWasmApp.targets`（ワークロード）から `Microsoft.NET.Sdk.WebAssembly.Browser.targets`（WASM SDK）に移行し、ワークロード非依存でもライブラリモードのトリミング設定が利用可能に
- **SelfContained デフォルト削除**: `WorkloadManifest.targets.in` の `browser-wasm` 向け `SelfContained=true` を廃止
- **PublishTrimmed デフォルト削除**: ワークロード側の `PublishTrimmed=true` デフォルトを廃止（WASM SDK で設定）
- **ワークロード必須条件の緩和**: `_WasmNativeWorkloadNeeded` ゲートから `_UsingBlazorOrWasmSdk` を削除し、`RID=browser-wasm` 設定だけではワークロード不要に
- **テストアセット簡略化**: `LazyLibrary.csproj` から不要な `UseMonoRuntime` を削除
- **テスト拡充**: ライブラリモードテストを 1 ケースから 4 ケースに拡充（`useWasmSdk` × `isPublish` マトリクス、すべて `no-workload` タグ付き、ビルド/パブリッシュの両方に対応）

#### パフォーマンスへの影響
影響なし

#### 関連Issue
[#123477](https://github.com/dotnet/runtime/issues/123477)

#### その他
- 公開 API の変更なし（SDK 設定調整のみ）
- ワークロード非インストール環境での WebAssembly ライブラリ開発がサポートされるようになる
