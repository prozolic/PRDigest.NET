### [#128684](https://github.com/dotnet/runtime/pull/128684) Arm64: Don't use GT_LEA for masks {#128684}
- 作成者: [@a74nh](https://github.com/a74nh)
- 作成日時: 2026年05月28日 08:55:40(UTC)
- マージ日時: 2026年05月31日 07:52:06(UTC)
- ラベル: <span style="background-color: #d4c5f9; color: #000000; display: inline-block; padding: 0 7px; font-size:12px; font-weight:500; line-height:18px; border-radius:2em; border:1px solid transparent; cursor:default;">area-CodeGen-coreclr</span> <span style="background-color: #c2e0c6; color: #000000; display: inline-block; padding: 0 7px; font-size:12px; font-weight:500; line-height:18px; border-radius:2em; border:1px solid transparent; cursor:default;">community-contribution</span>
#### 概要
ARM64 SVE における mask/SIMD ロード/ストア命令のアドレスモード生成を制限し、LEA ノードの使用を回避する変更です。従来のスケール可能な即値オフセット処理を削除し、lowering フェーズで address mode 生成を拒否することで、emitter の複雑性を軽減しています。

#### 変更内容
- **src/coreclr/jit/lower.cpp**: ARM64 で `TYP_MASK`/`TYP_SIMD` 型の親ノードに対して address mode 生成をスキップする処理を追加
- **src/coreclr/jit/emitarm64sve.cpp**: SVE ldr/str emitter の即値スケーリングワークアラウンドを削除し、SP をベースレジスタとして許可
- **src/coreclr/jit/emitarm64.h** / **src/coreclr/jit/emitarm64.cpp**: `emitIns_valid_imm_for_scaled_sve_ldst_offset` ヘルパー関数の宣言・実装を削除

#### パフォーマンスへの影響
影響なし。本変更は address mode 生成ロジックを lowering フェーズに移行させるもので、emitter の即値スケーリング処理削除による複雑性軽減が目的です。

#### 関連Issue
[#127605](https://github.com/dotnet/runtime/issues/127605)

#### その他
将来的にスケーラブルベクトル/mask に対応する際の `genCreateAddrMode()` 書き直しに向けた準備段階です。emitter は 9-bit 符号付き即値のバリデーションのみを行うように簡略化されています。
