- 作成日時: 2026年05月07日 13:47:00(UTC)
- マージ日時: 2026年05月20日 12:32:23(UTC)
- ラベル: <a style="text-decoration:none;" href="../../labels/area-CodeGen-coreclr/index.html"><span style="background-color: #d4c5f9; color: #000000; display: inline-block; padding: 0 7px; font-size:12px; font-weight:500; line-height:18px; border-radius:2em; border:1px solid transparent;">area-CodeGen-coreclr</span></a> <a style="text-decoration:none;" href="../../labels/community-contribution/index.html"><span style="background-color: #c2e0c6; color: #000000; display: inline-block; padding: 0 7px; font-size:12px; font-weight:500; line-height:18px; border-radius:2em; border:1px solid transparent;">community-contribution</span></a>
#### 概要
JITコンパイラの条件式最適化を改善し、`SELECT(cond, cns, cns)`パターン（条件は異なるが真偽両ケースで同じ定数値を返す）を直接その定数値に変換します。これにより、不要な条件分岐と比較演算を削除し、生成されるアセンブリコードを簡潔化します。

例：`0.0.Equals(value)`のような比較は、条件の結果に関わらず定数を返す場合、その定数に直接置換されます。

#### 変更内容
- **src/coreclr/jit/ifconversion.cpp**: `SELECT`式の最適化ロジックに4行追加。同じ定数値を返す`SELECT(cond, cns, cns)`パターンを検出し、定数値に直接変換する処理を実装。

#### パフォーマンスへの影響
改善あり。生成されるアセンブリコードサイズが縮小（提示例では約32バイトから19バイトへ削減）され、不要な分岐とCMOV命令が排除されます。

**ベンチマーク（提示例）:**
- 基準版：複数のCMOV命令と分岐が含まれる
- 最適化版：単純な比較命令とMOVZX命令のみで処理完了

#### 関連Issue
なし

#### その他
- コメントで「`gtFoldExprConditional`でも同様の処理が可能」との記述があり、他の最適化パスとの関連性が存在する可能性があります
- 内部実装（JIT最適化ロジック）の改善であり、公開APIへの影響はありません

### [#127503](https://github.com/dotnet/runtime/pull/127503) Add configurable VariableNameTransformation to EnvironmentVariables config {#127503}
- 作成者: [@svick](https://github.com/svick)
- 作成日時: 2026年04月28日 12:54:46(UTC)
- マージ日時: 2026年05月20日 23:46:13(UTC)
- ラベル: <a style="text-decoration:none;" href="../../labels/area-Extensions-Configuration/index.html"><span style="background-color: #d4c5f9; color: #000000; display: inline-block; padding: 0 7px; font-size:12px; font-weight:500; line-height:18px; border-radius:2em; border:1px solid transparent;">area-Extensions-Configuration</span></a>
#### 概要
環境変数の設定キー変換をカスタマイズできる機能を追加します。デフォルトではアンダースコア2つ(`__`)をコロン(`:`)に変換しますが、新たに`VariableNameTransformation`デリゲートで任意の変換ロジックを指定できるようになります。また、ドット(`.`)を含むキーを環境変数から設定する場合用に`ColonAndDotTransformation`プリセットが提供されます。

```csharp
// アンダースコア3つで`.`、2つで`:`に変換
builder.AddEnvironmentVariables("Logging__", 
    EnvironmentVariablesConfigurationSource.ColonAndDotTransformation);
```

#### 変更内容
- **EnvironmentVariablesExtensions.cs**: `AddEnvironmentVariables(string? prefix, Func<string, string>? variableNameTransformation)`オーバーロード追加
- **EnvironmentVariablesConfigurationSource.cs**: 
  - `VariableNameTransformation`プロパティ（カスタム変換関数の設定用）
  - `DefaultTransformation`プロパティ（既存の`__`→`:`変換）
  - `ColonAndDotTransformation`プロパティ（`___`→`.`、`__`→`:`の変換）
- **EnvironmentVariablesConfigurationProvider.cs**: 変換ロジックの実装（プレフィックスも変換済み形式で指定）
- **EnvironmentVariablesTest.cs**: 13個の新規テストケース（2つの組み込み変換の動作検証、カスタム変換の適用確認、プレフィックス変換の検証など）
- **ref/定義ファイル**: 公開API定義の更新

#### パフォーマンスへの影響
- `ColonAndDotTransformation`は.NETでは`ValueStringBuilder` + `stackalloc char[256]`を使用して低レイテンシ処理を実現
- .NET Frameworkでは`StringBuilder`を使用（条件付きコンパイル`#if NET`で分岐）
- 新規パッケージ依存なし
- デフォルト変換時は既存と同等のパフォーマンス

#### 関連Issue
[#125137](https://github.com/dotnet/runtime/issues/125137)

#### その他
- プレフィックスはアクティブな変換前の形式（`"Logging__"`）で指定する必要があります。変換後の形式（`"Logging:"`）ではなく、`Prefix`プロパティのドキュメントおよび関連するオーバーロードで明記されています
- API設計は既承認のコメントに準拠しています

### [#125298](https://github.com/dotnet/runtime/pull/125298) JIT: Add debug consistency checks for `GT_RETURN` nodes {#125298}
- 作成者: [@BoyBaykiller](https://github.com/BoyBaykiller)
- 作成日時: 2026年03月08日 00:11:23(UTC)
- マージ日時: 2026年05月20日 12:30:48(UTC)
- ラベル: <a style="text-decoration:none;" href="../../labels/area-CodeGen-coreclr/index.html"><span style="background-color: #d4c5f9; color: #000000; display: inline-block; padding: 0 7px; font-size:12px; font-weight:500; line-height:18px; border-radius:2em; border:1px solid transparent;">area-CodeGen-coreclr</span></a> <a style="text-decoration:none;" href="../../labels/community-contribution/index.html"><span style="background-color: #c2e0c6; color: #000000; display: inline-block; padding: 0 7px; font-size:12px; font-weight:500; line-height:18px; border-radius:2em; border:1px solid transparent;">community-contribution</span></a>
#### 概要
JITコンパイラの中間表現（IR）一貫性チェックを強化し、`GT_RETURN`ノードが正しく配置されていることを検証するデバッグアサーションを追加します。`GT_RETURN`は`BBJ_RETURN`ブロックにのみ存在し、かつそのブロック内の最後のステートメントである必要があります。

#### 変更内容
- **src/coreclr/jit/fgdiagnostic.cpp**: `fgDebugCheckStmtsList`関数を拡張し、以下の2つのアサーションを追加
  1. `BBJ_RETURN`以外のブロックは`GT_RETURN`ノードを含まないことを検証
  2. `BBJ_RETURN`ブロック内の`GT_RETURN`がブロックの最後のステートメントであることを検証

#### パフォーマンスへの影響
デバッグビルド時のみ有効なチェックのため、リリースビルドのパフォーマンスへの影響なし。デバッグビルド時はIR検証のため若干のオーバーヘッドが増加。

#### 関連Issue
[#125072](https://github.com/dotnet/runtime/issues/125072)

#### その他
このアサーションは、PR #125072がマージされた後、理想的には発火しないことが期待されています。IR一貫性の維持に関する防御的プログラミング的なアプローチです。
