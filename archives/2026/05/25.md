### [#128518](https://github.com/dotnet/runtime/pull/128518) \[cDAC\] Restore Frame-walk fallback in Thread.GetContext {#128518}
- 作成者: [@Copilot](https://github.com/apps/copilot-swe-agent)
- 作成日時: 2026年05月23日 05:34:26(UTC)
- マージ日時: 2026年05月25日 05:55:24(UTC)
- ラベル: <a style="text-decoration:none;" href="../../labels/area-Diagnostics-cdac/index.html"><span style="background-color: #d4c5f9; color: #000000; display: inline-block; padding: 0 7px; font-size:12px; font-weight:500; line-height:18px; border-radius:2em; border:1px solid transparent;">area-Diagnostics-cdac</span></a>
#### 概要
cDAC（Compact Data Access Component）の `Thread.GetContext` メソッドに Frame-walk フォールバック機能を復元しました。データターゲットが `GetThreadContext` を実装していない場合、スレッドの Frame チェーンをウォークしてスレッドコンテキストを導出します。これは以前削除されたネイティブ側の実装と同等の処理を cDAC に取り込むものです。

#### 変更内容
- **`Thread_1.cs`**: `TryGetThreadContext` が失敗した場合、スレッドの Frame チェーンをウォークしてコンテキストを復元
  - `InterpreterFrame` の場合は、既存の `UpdateContextFromCurrentFrame` を経由して top `InterpMethodContextFrame` からコンテキストを取得
  - その他の Frame については、クリアされたコンテキストを更新し、SP と PC がともに 0 でない最初のフレーム（`RedirectedThreadFrame`、`InlinedCallFrame`、`DynamicHelperFrame` など）を採用し、`RawContextFlags = FullContextFlags` を設定
  - usable なコンテキストを提供するフレームがない場合は、ゼロ化されたコンテキストを返す（スレッドが managed コード実行中でない）

- **`docs/design/datacontracts/Thread.md`**: `GetContext` のフォールバック動作を文書化

#### パフォーマンスへの影響
影響なし

#### 関連Issue
[#128518](https://github.com/dotnet/runtime/issues/128518)

#### その他
- 本変更は内部実装（cDAC）に対するもので、公開API の変更ではありません
- ネイティブ側の対応は別途 PR で実施されています
- Frame-walk フォールバックは、OS スレッドコンテキストが入手不可能なデータターゲットに対する互換性復元です
