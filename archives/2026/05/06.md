### [#127823](https://github.com/dotnet/runtime/pull/127823) Add StringBuilder.MoveChunks\(StringBuilder\) API {#127823}
- 作成者: [@Copilot](https://github.com/apps/copilot-swe-agent)

- マージ日時: 2026年05月06日 16:01:20(UTC)
- ラベル: <span style="background-color: #d4c5f9; color: #000000; display: inline-block; padding: 0 7px; font-size:12px; font-weight:500; line-height:18px; border-radius:2em; border:1px solid transparent; cursor:default;">area-System.Runtime</span>
#### 概要
`StringBuilder.MoveChunks(StringBuilder source)` という新しい公開APIを追加します。このメソッドは、ソースの`StringBuilder`のチャンク チェーンを新しい`StringBuilder`インスタンスにO(1)で転送し、ソースをドレイン（空）にします。これにより、Roslyn等のコンシューマーがコピーなしで不変ビューを保持しながら、元の`StringBuilder`を解放できます。

```csharp
namespace System.Text;

public partial class StringBuilder
{
    public static StringBuilder MoveChunks(StringBuilder source);
}
```

#### 変更内容
- **`StringBuilder.cs`** - `MoveChunks`メソッドを実装。既存のプライベートコピーコンストラクタを利用して、ソースの`m_ChunkChars`、`m_ChunkPrevious`、`m_ChunkLength`、`m_ChunkOffset`、`m_MaxCapacity`を新しいインスタンスに転送。ソースはドレイン後、`m_ChunkChars`を`Array.Empty<char>()`に設定し、空だが完全に利用可能な状態を維持。引数が`null`の場合は`ArgumentNullException`をスロー。
- **`System.Runtime.cs`** - リファレンスアセンブリに新しい公開シグネチャを追加。
- **`StringBuilderTests.cs`** - null引数、空ソース、単一チャンク、マルチチャンク チェーン、ドレイン済みソースへの再呼び出し、ドレイン後の利用可能性、`MaxCapacity`保持などをカバーする119行のテストを追加。

#### パフォーマンスへの影響
影響なし。転送操作はO(1)で実行でき、内部フィールドの参照付け替えのみです。既存コード パスで到達可能なフィールドのみを操作するため、オーバーヘッドはありません。

#### 関連Issue
[#97570](https://github.com/dotnet/runtime/issues/97570)、dotnet/roslyn#61326

#### その他
新しいAPIの追加のみで、既存メンバーの動作変更はありません。リスクレベルは低く、破壊的変更や後方互換性への影響はありません。
