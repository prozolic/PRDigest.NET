### [#128389](https://github.com/dotnet/runtime/pull/128389) Replace small stackallocs with collection literals {#128389}
- 作成者: [@EgorBo](https://github.com/EgorBo)
- 作成日時: 2026年05月20日 00:26:21(UTC)
- マージ日時: 2026年05月21日 19:40:07(UTC)
- ラベル: <a style="text-decoration:none;" href="../../labels/area-Meta/index.html"><span style="background-color: #d4c5f9; color: #000000; display: inline-block; padding: 0 7px; font-size:12px; font-weight:500; line-height:18px; border-radius:2em; border:1px solid transparent;">area-Meta</span></a> <a style="text-decoration:none;" href="../../labels/reduce-unsafe/index.html"><span style="background-color: #0F6E51; color: #ffffff; display: inline-block; padding: 0 7px; font-size:12px; font-weight:500; line-height:18px; border-radius:2em; border:1px solid transparent;">reduce-unsafe</span></a>
#### 概要
非常に小さい `stackalloc` バッファを C# collection expressions（`[0, 0, 0, 0]` など）に置き換え、それらの `stackalloc` のためだけに必要だった `unsafe` 修飾子を削除するリファクタリングです。スタックベースのバッファ動作は維持しながら、コード安全性を向上させます。

```csharp
// 変更前
unsafe void Example()
{
    Span<int> buffer = stackalloc int[4];
}

// 変更後
void Example()
{
    Span<int> buffer = [0, 0, 0, 0];
}
```

#### 変更内容
- **26ファイルにおける stackalloc の置き換え**: 1～4要素の小規模な `stackalloc` を collection expressions に置き換え
  - `stackalloc int[4]` → `[0, 0, 0, 0]`
  - `stackalloc byte[1]` → `[0]`
  - `stackalloc char[2]` → `['\0', '\0']` など
- **unsafe 修飾子の削除**: `stackalloc` 専用だった `unsafe` コンテキストを削除
- **影響範囲**: System.Text.RegularExpressions、System.Text.Json、System.Security.Cryptography、System.Private.CoreLib、System.IO.Pipelines など複数のライブラリ

#### パフォーマンスへの影響
影響なし。Collection expressions は C# コンパイラにより `stackalloc` と同等のスタック割り当てに最適化されるため、実行時パフォーマンスおよびメモリ配置に変化なし。

#### 関連Issue
なし

#### その他
- **内部実装の変更**: 公開 API に直接的な影響はなし
- **互換性**: 互換性への影響なし（シグネチャ変更なし）
- **コード品質向上**: `unsafe` コンテキストの削除により、より安全で検証可能なコードとなる
