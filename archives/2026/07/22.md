#### 概要
WebAssembly環境において、R2R（ReadyToRun）コンパイル済みコードから呼び出されるマネージドプログラムのエントリーポイントが未初期化のPortableEntryPoint（PEP）に解決される場合、間接呼び出しが失敗してトラップするか、全プログラムのインタープリータフォールバックが発生する問題を修正します。`RunMainInternal`内でエントリーポイントアドレスを取得した直後に`MethodDesc::EnsurePortableEntryPointIsCallableFromR2R`を呼び出し、PEPをネイティブR2Rコードまたは正しく型付けされたインタープリータサンク（thunk）に確実に発行します。

```cpp
// RunMainInternal内での追加コード
pParam->pFD->EnsureActive();
PCODE entryPoint = pParam->pFD->GetSingleCallableAddrOfCode();
#ifdef FEATURE_PORTABLE_ENTRYPOINTS
MethodDesc::EnsurePortableEntryPointIsCallableFromR2R(entryPoint);
#endif // FEATURE_PORTABLE_ENTRYPOINTS
```

#### 変更内容
- **src/coreclr/vm/assembly.cpp**：`RunMainInternal`内でエントリーポイント取得直後に`EnsurePortableEntryPointIsCallableFromR2R`呼び出しを追加（+6行）

#### パフォーマンスへの影響

- **修正後**：String..ctor R2R実行回数 7、Memmove R2R実行回数 8、TryPublishR2R FAILED件数 0

修正により、R2Rネイティブコード実行が確実になり、インタープリータフォールバック診断が完全に消失します。

#### 関連Issue
[#130386](https://github.com/dotnet/runtime/issues/130386)、[#130841](https://github.com/dotnet/runtime/issues/130841)

#### その他

#### 概要

- `[Field]`、`[FieldAddress]`、`[InstanceDataStart]`、`[RawOffset]`属性を持つすべてのプロパティに`partial`修飾子を追加（179ファイル）
- ジェネレータが自動プロパティの代わりにゲッターボディを生成できるようにするため

**2. ジェネレータの遅延読み込み実装**
- 各フィールドに対して、バッキングフィールド + 読み込みフラグを持つ遅延読み込みパーシャルゲッターを生成
- 最初のアクセス時にターゲットから読み込みしメモ化（キャッシング）
- オプショナル（`T?`）フィールド：ディスクリプタに欠落している場合、`null`を返す
- 必須フィールド：読み込み時に失敗（`VirtualReadException`または`InvalidOperationException`）
- 検証用に新インターフェース`IReadableData.EnsureAllFieldsRead()`を追加（全フィールド読み込みを強制）

**3. `ISHash`ルックアップの戻り値を可nullable化**
- `ISHash.LookupSHash`が`TEntry?`を返すように変更（キーが欠落時は`null`）
- `ITraits.Null()`センチネルを廃止
- `DynamicILBlobEntry`が単純な読み取り専用遅延ビューに簡潔化

**4. テスト**
- `Microsoft.Diagnostics.DataContractReader.DataGenerator.Tests`：35件パス
- `Microsoft.Diagnostics.DataContractReader.Tests`：2693件パス、0件失敗

#### パフォーマンスへの影響

**メモリ**：遅延評価により、各フィールドに対してバッキングフィールド＋読み込みフラグが必要になるため、インスタンスあたりのメモリ使用量が若干増加する可能性があります。

**実行速度**：構築時の読み込み処理が廃止されるため、`IData`インスタンスの構築速度が改善されます。一方、初回フィールドアクセス時のオーバーヘッド（読み込み + メモ化）が生じます。全フィールドへのアクセスを行う場合、全体的なパフォーマンスは同等またはわずかに低下する可能性があります。

**メリット**：コンストラクタ失敗がなくなるため、部分的に読み取り可能な構造でも`IData`インスタンスが生成でき、実際に使用されないフィールドの欠落エラーを回避できます。
