#### 概要
`Guid`、`Int128`、`UInt128`型をbitwise equatableとして扱い、`SequenceEqual`がメモリ比較の最適化（memcmp unrolling）を利用できるようにしました。これにより、`Guid.Equals`をメモリセーフなコードで実装できるようになります。

```csharp
void Foo(Guid g1, Guid g2)
{
    if (((Span<Guid>)[g1]).SequenceEqual((Span<Guid>)[g2]))
    {
        Console.WriteLine("Equal");
    }
}
```

上記のコード例では、関数呼び出しからSIMD命令（vmovups、vpcmpnequq）を使った直接メモリ比較に最適化されます。

#### 変更内容
- **src/coreclr/tools/Common/TypeSystem/IL/Stubs/RuntimeHelpersIntrinsics.cs**: `RuntimeHelpers.IsBitwiseEquatable<T>()`のIL生成時に、`Guid`、`Int128`、`UInt128`をbitwise equatableな既知型のホワイトリストに追加
- **src/coreclr/vm/jitinterface.cpp**: EEがILサブスティテューション時にこれらの型を認識するための実装を追加
- **src/coreclr/jit/instr.cpp**: JitDisasmで使用される整数比較の疑似名を修正

#### パフォーマンスへの影響
改善あり。`Guid`の比較が関数呼び出しベースからSIMD命令（vmovups、vpcmpnequqなど）を用いたメモリ直接比較に最適化され、呼び出しオーバーヘッドと命令数が削減されます。提供されたコード例では、`SpanHelpers:SequenceEqual`への関数呼び出しが排除され、より効率的なマシンコードが生成されます。

#### 関連Issue
[#130644](https://github.com/dotnet/runtime/issues/130644)

#### その他
