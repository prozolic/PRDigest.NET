### [#130332](https://github.com/dotnet/runtime/pull/130332) Arm64: use FEAT_CSSC cnt/ctz for PopCount and TrailingZeroCount {#130332}
- 作成者: [@EgorBo](https://github.com/EgorBo)
- 作成日時: 2026年07月08日 07:14:07(UTC)
- マージ日時: 2026年07月12日 12:03:32(UTC)
- ラベル: <a style="text-decoration:none;" href="../../labels/area-CodeGen-coreclr/index.html"><span style="background-color: #d4c5f9; color: #000000; display: inline-block; padding: 0 7px; font-size:12px; font-weight:500; line-height:18px; border-radius:2em; border:1px solid transparent;">area-CodeGen-coreclr</span></a>
#### 概要
Arm64アーキテクチャにおいて、FEAT_CSSC（ARMv8.8以上対応）の検出と統合を実装し、`BitOperations.PopCount`および`BitOperations.TrailingZeroCount`をスカラ`cnt`/`ctz`命令で最適化します。対応CPUでは、ベクトルレジスタとのラウンドトリップを回避し、特にPopCount(ulong)で大幅なパフォーマンス向上を実現します。

```csharp
// 最適化の対象
static int TestPOPCNT(uint value) => BitOperations.PopCount(value);
static int TestPOPCNT64(ulong value) => BitOperations.PopCount(value);
static int TestTZC(uint value) => BitOperations.TrailingZeroCount(value);
static int TestTZC64(ulong value) => BitOperations.TrailingZeroCount(value);
```

#### 変更内容
- **CPU機能検出**: minipalでLinux HWCAPおよびApple sysctlを経由してFEAT_CSSCを検出し、VM CPU flagsに反映
- **ISA統合**: InstructionSet_CsscをJIT内部で定義し、Ready2Runメタデータに追加（R2R bit 93）
- **コード生成最適化**:
  - PopCount: ベクトル演算（movi+ins+cnt+addv+umov）から`cnt w0, w0`/`cnt x0, x0`へ簡略化
  - TrailingZeroCount: `rbit+clz`の2命令から`ctz w0, w0`/`ctz x0, x0`の1命令へ簡略化
- **LSRA/エミッター更新**: CSSC利用時にSIMDテンポラリレジスタを不要化し、スカラ命令エンコーディングを追加
- **設定ゲート**: `EXTERNAL_EnableArm64Cssc`設定スイッチで制御可能
- **JIT/EE互換性**: インターフェース版GUIDを更新

#### パフォーマンスへの影響
Apple M5 Proでのベンチマーク結果（1024要素配列を対象）:

| メソッド | 現在 | CSSC適用後 | 改善率 |
|---------|------|----------|--------|
| PopCount(uint) | 271.9 ns | 247.5 ns | 0.91倍（9%改善） |
| PopCount(ulong) | 1,568.5 ns | 247.7 ns | **0.16倍（84%改善）** |
| TrailingZeroCount(uint) | 255.4 ns | 246.3 ns | 0.96倍（4%改善） |
| TrailingZeroCount(ulong) | 255.8 ns | 243.4 ns | 0.95倍（5%改善） |

PopCount(ulong)の大幅な改善はGPR⟷SIMDラウンドトリップ回避に起因。サーバーArm64ハードウェアでは異なる結果の可能性あり。

#### 関連Issue
なし

#### その他
- Apple M5/M4では完全対応、Helix環境（Apple M4）ではまだCSCSを非サポートのため、M5以上またはNeoverse N4+/V4+搭載ハードウェアで全機能テスト可能
- 今後CSSC追加命令対応の可能性：`abs`、`smax`など
