### [#129636](https://github.com/dotnet/runtime/pull/129636) Disable scope info for wasm targets {#129636}
- 作成者: [@AndyAyersMS](https://github.com/AndyAyersMS)
- 作成日時: 2026年06月19日 20:10:07(UTC)
- マージ日時: 2026年06月21日 14:47:12(UTC)
- ラベル: <a style="text-decoration:none;" href="../../labels/area-CodeGen-coreclr/index.html"><span style="background-color: #d4c5f9; color: #000000; display: inline-block; padding: 0 7px; font-size:12px; font-weight:500; line-height:18px; border-radius:2em; border:1px solid transparent;">area-CodeGen-coreclr</span></a>
#### 概要
WebAssembly(WASM)ターゲットではネイティブデバッガが存在せず、仮想レジスタがICorDebugInfoで表現できないため、WASMビルドでスコープ情報(compScopeInfo)の生成を無効化する変更です。あわせてリターン値の収集もこのフラグで制御するようにします。

#### 変更内容
- **src/coreclr/jit/compiler.cpp**: WASM ターゲット(`TARGET_WASM`)でのみ`compScopeInfo`を無効化するガード処理を追加（+7行）
- **src/coreclr/jit/codegencommon.cpp**: リターン値の収集をスコープ情報フラグの制御下に配置（+2/-2行）

#### パフォーマンスへの影響
WASM ターゲットでスコープ情報生成が不要になることで、JIT コンパイル時間と生成コードサイズがわずかに削減される可能性があります。

#### 関連Issue
[#129630](https://github.com/dotnet/runtime/issues/129630)

#### その他
- WASM ターゲットではネイティブデバッガが存在しないため、スコープ情報の生成は不要です
- 内部実装（JIT コンパイラのデバッグ情報生成処理）の変更であり、公開 API への影響なし
- GitHub Copilot の支援により作成されたコントリビューション

### [#129614](https://github.com/dotnet/runtime/pull/129614) \[mono\] LLVM backend correctness, NaN-safe FP narrow, and test annotations {#129614}
- 作成者: [@pavelsavara](https://github.com/pavelsavara)
- 作成日時: 2026年06月19日 11:08:47(UTC)
- マージ日時: 2026年06月21日 12:45:11(UTC)
- ラベル: <a style="text-decoration:none;" href="../../labels/area-Build-mono/index.html"><span style="background-color: #d4c5f9; color: #000000; display: inline-block; padding: 0 7px; font-size:12px; font-weight:500; line-height:18px; border-radius:2em; border:1px solid transparent;">area-Build-mono</span></a>
#### 概要

Emscripten 5.0.6 / LLVM 23へのアップグレードに向けたMono LLVM バックエンドの事前準備PR。LLVM 23の厳格な定数エンコーディングとより積極的な浮動小数点変換の認識に対応させる。すべてのLLVM 23対応コードは`LLVM_API_VERSION`ガードされており、現在のLLVM 19では動作的に同等であり、アップグレード前に安全にマージ可能。

#### 変更内容

**Mono LLVM バックエンド (`src/mono/mono/mini/`)**

- **LLVMContextRef のスレッド化**: `mono_llvm_create_constant_data_array`、`mono_llvm_get_ptr_type`、構造体リターン/byval属性ヘルパー、デバッグロケーション作成、GCセーフポイントポーリング、イントリンシック登録が、グローバルコンテキスト（`LLVMGetGlobalContext()`）の代わりにモジュール/関数/命令のコンテキストを使用するよう更新。複数コンテキスト使用時の正確性に必須。
- **LLVM 21 イントリンシック宣言API採用**: `LLVM_API_VERSION >= 2100`で`Intrinsic::getOrInsertDeclaration`を使用、旧バージョンでは`getDeclaration`にフォールバック。
- **負の整数定数のマスキング**: `const_int8`と`const_int32`が`LLVMConstInt`呼び出し前に値をターゲット幅にマスク。LLVM 23以降、`SignExtend`が`FALSE`の場合に上位ビットが保持され、`ZExt`結果が誤って折り畳まれることを防止（例：`zext i8 -1 to i64`が`-1`ではなく`255`を返す）。
- **`LLVMConstAllOnes`の活用**: div-check `-1`比較と`il_state`初期化で、`LLVMConstInt(..., -1, FALSE)`の代わりに使用。
- **符号付き即値の符号拡張**: `OP_ICOMPARE_IMM`で`LLVMConstInt(IntPtrType(), ins->inst_imm, TRUE)`を使用。
- **NaN保持浮動小数点ナローイング**: 新しい`emit_fpnarrow`/`quiet_nan_for_fptype`ヘルパーがNaN入力を明示的に検出し、目的地型の正規静かNaNを選択。arm64などのターゲットでハードウェア`fptrunc`がNaN仮数を破棄して±Infinityに変換するケースに対応（`LLVM_API_VERSION >= 2300`でガード）。
- **非推奨警告の抑止**: `mini-llvm.c`上部で`-Wdeprecated-declarations`を抑止。LLVM-Cグローバルコンテキト型アクセサ（`LLVMInt32Type`等）が新LLVM版で非推奨化されているため。

**ワークロードテスト対象 (`WorkloadTesting.Core.targets`)**

- `ci.dot.net/public` → `ci.dot.net/dev`の書き換えをMSBuild経由ではなく、プラットフォーム標準ツール（Unix: `sed`、Windows: PowerShell）で実行。MSBuild文字列/アイテムリスト解析の落とし
