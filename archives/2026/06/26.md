  - `AnalyzeIteration` の遅延呼び出しとキャッシング機構を追加し、対象ループのみが分析コストを負担するように最適化
  - ダンプメッセージとコメントを更新

  - Code size: +0.18%、PerfScore overall: −0.04%、TP (release): +0.46%

**注記：** TP コスト増加は主に実際の inversion フェーズ実行によるもので、lazy `AnalyzeIteration` 呼び出し自体は小さい割合です。

### [#128741](https://github.com/dotnet/runtime/pull/128741) Sanitize control/format characters in console logger output across all formatters {#128741}
- 作成者: [@Copilot](https://github.com/apps/copilot-swe-agent)
- 作成日時: 2026年05月29日 04:44:55(UTC)
- マージ日時: 2026年06月26日 15:21:17(UTC)
- ラベル: <a style="text-decoration:none;" href="../../labels/area-Extensions-Logging/index.html"><span style="background-color: #d4c5f9; color: #000000; display: inline-block; padding: 0 7px; font-size:12px; font-weight:500; line-height:18px; border-radius:2em; border:1px solid transparent;">area-Extensions-Logging</span></a>
#### 概要
コンソールロギングで信頼できないコントロール文字が出力されるセキュリティ脆弱性を修正するPRです。`SimpleConsoleFormatter`と`SystemdConsoleFormatter`で Unicode `Cc`（制御文字）/`Cf`（形式文字）を `\uXXXX` 形式にエスケープして出力することで、ターミナルエスケープシーケンスの実行やあいまいな出力を防止します。

#### 変更内容
- **ConsoleControlCharacterSanitizer.cs（新規）**: 制御文字・形式文字を検出してエスケープする共有サニタイザーを追加
- **SimpleConsoleFormatter.cs**: メッセージ/例外/カテゴリ/スコープテキストを出力前にサニタイズ
- **SystemdConsoleFormatter.cs**: メッセージ/例外/カテゴリ/スコープテキストを出力前にサニタイズ
- **JsonConsoleFormatter.cs**: char値の状態プロパティ処理を調整（サニタイザーは適用されていない）
- **Microsoft.Extensions.Logging.Console.csproj**: `ValueStringBuilder`への依存を追加
- **ConsoleFormatterTests.cs**: サニタイズ動作の単体テストを追加（現在は非JSONフォーマッターのみ対象）

#### パフォーマンスへの影響
サニタイズ処理により若干のメモリ割り当てと処理時間が増加します。`ValueStringBuilder`の使用により割り当て最小化に配慮されています。影響なし（セキュリティ上の重要な修正であり、パフォーマンス上の問題報告なし）

#### 関連Issue
[#128727](https://github.com/dotnet/runtime/issues/128727)

#### その他
- **動作変更**: コントロール/形式文字は出力時に `\uXXXX` 形式にエスケープされます。既存のログ解析スクリプトがこれに依存する場合は影響を受ける可能性があります。
- JSONフォーマッターはサニタイザー適用対象外のため、JSON出力内の制御文字は別途対策が必要な可能性があります。
