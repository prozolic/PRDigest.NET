#### 概要
メモリおよびテキストベースの型に対する標準化されたストリームラッパーを新しい公開シール型として追加します。`StringStream`、`ReadOnlyMemoryStream`、`WritableMemoryStream`（CoreLib）、および`ReadOnlySequenceStream`（System.Memory）が導入され、テキスト・メモリデータをStream APIで統一的に扱えるようになります。

サンプルコード:
```csharp
// StringStream: 文字列をストリーム化してエンコード
var stream = new StringStream("Hello, World!", Encoding.UTF8);
byte[] buffer = new byte[100];
int bytesRead = stream.Read(buffer, 0, buffer.Length);

// ReadOnlyMemoryStream: バイト配列の読み取り専用ストリーム
ReadOnlyMemory<byte> data = new byte[] { 1, 2, 3 };
var romStream = new ReadOnlyMemoryStream(data);

// WritableMemoryStream: 固定容量の読み書き可能ストリーム
Memory<byte> buffer = new byte[256];
var wmStream = new WritableMemoryStream(buffer);
wmStream.Write(new byte[] { 1, 2, 3 }, 0, 3);

// ReadOnlySequenceStream: ReadOnlySequence<byte>のストリーム化
ReadOnlySequence<byte> sequence = ...;
var rosStream = new ReadOnlySequenceStream(sequence);
```

#### 変更内容
- **StringStream**（CoreLib）：文字列またはReadOnlyMemory<char>を動的にエンコードする非シーク型の読み取り専用ストリーム。エンコーディング指定が必須でBOMを出力しない。
- **ReadOnlyMemoryStream**（CoreLib）：ReadOnlyMemory<byte>上のシーク可能な読み取り専用ストリーム。MemoryStreamから派生。
- **WritableMemoryStream**（CoreLib）：Memory<byte>上の固定容量のシーク可能な読み書きストリーム。MemoryStreamから派生。
- **ReadOnlySequenceStream**（System.Memory）：ReadOnlySequence<byte>上のシーク可能な読み取り専用ストリーム。
- 参照アセンブリ（System.Runtime、System.Memory）の更新
- 内部呼び出し箇所（ReadOnlyMemoryContent、XmlPreloadedResolver、JsonXmlDataContract）の新型への置き換え
- 全4型の準拠テストおよび動作テスト

#### パフォーマンスへの影響
StringStreamはバッファリングなしで呼び出し元バッファに直接エンコードするため、メモリ効率が良好。ReadOnlyMemoryStreamおよびWritableMemoryStreamはMemoryStreamベースで、既存実装と同等のパフォーマンス特性を継承。影響なし（改善による最適化）。
