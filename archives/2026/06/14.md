#### 概要
CoreCLR WebAssembly ブラウザローダーが `applicationCulture` を ICU シャードの選択にのみ使用していた問題を修正しました。`LANG` 環境変数を `"<applicationCulture>.UTF-8"` に設定することで、ランタイムのデフォルトロケールが `applicationCulture` と一致するようになり、Mono の実装に合わせます。これにより、非デフォルトカルチャで起動した Blazor WebAssembly アプリケーションで発生していた `InvalidOperationException` ("culture change is not supported") が解決されます。

```typescript
if (culture && loaderConfig.environmentVariables!["LANG"] === undefined) {
    loaderConfig.environmentVariables!["LANG"] = `${culture}.UTF-8`;
}
```

#### 変更内容
- **src/native/libs/Common/JavaScript/loader/icu.ts**: `applicationCulture` の確定後に `LANG` 環境変数を設定するロジックを追加（+4行）
  - ユーザーが既に `LANG` を指定している場合はオーバーライドしない保守的な実装
  - 環境変数の適用は `host/index.ts::setupEmscripten` で `dotnetInitializeModule` 中に行われるため、タイミングは正しい

#### パフォーマンスへの影響
影響なし

#### 関連Issue
関連 PR: [aspnetcore#66331](https://github.com/dotnet/aspnetcore/pull/66331)

#### その他
- **互換性**: 破壊的変更なし。ユーザーが明示的に `LANG` を設定している場合はその設定が優先される
- **動作変更**: `applicationCulture=fr-FR` で起動した場合、修正前は `CurrentCulture` が `en-US` にフォールバックしていたのに対し、修正後は `fr-FR` で一貫性が保たれます
- Mono の `src/mono/browser/runtime/loader/config.ts` と同じアプローチを採用し、CoreCLR-WASM と Mono のグローバライゼーション動作を統一
