### [#128619](https://github.com/dotnet/runtime/pull/128619) Reduce contention in UnwindInfoTable seen during high volume LCG method creation {#128619}
- 作成者: [@eduardo-vp](https://github.com/eduardo-vp)
- 作成日時: 2026年05月27日 03:36:23(UTC)
- マージ日時: 2026年06月05日 06:08:56(UTC)
- ラベル: <a style="text-decoration:none;" href="../../labels/os-windows/index.html"><span style="background-color: #fef2c0; color: #000000; display: inline-block; padding: 0 7px; font-size:12px; font-weight:500; line-height:18px; border-radius:2em; border:1px solid transparent;">os-windows</span></a> <a style="text-decoration:none;" href="../../labels/tenet-performance/index.html"><span style="background-color: #c2e0c6; color: #000000; display: inline-block; padding: 0 7px; font-size:12px; font-weight:500; line-height:18px; border-radius:2em; border:1px solid transparent;">tenet-performance</span></a> <a style="text-decoration:none;" href="../../labels/area-VM-coreclr/index.html"><span style="background-color: #d4c5f9; color: #000000; display: inline-block; padding: 0 7px; font-size:12px; font-weight:500; line-height:18px; border-radius:2em; border:1px solid transparent;">area-VM-coreclr</span></a>
#### 概要
大量のLCG（Lightweight Code Generation）メソッド作成時における`UnwindInfoTable`でのロック競合を削減する最適化です。.NET 10で確認された性能低下（.NET 9比で169%遅化）に対応し、グローバルロックをテーブル単位のロックに変更、ロック取得の条件付き化、バイナリサーチの導入などにより、LCGストレスベンチマーク（300K メソッド）では.NET 10比で44%改善を実現しています。

#### 変更内容
- **LCGMethodResolver**: `m_DynamicCodePointers`に初期ポインタを追加し、ほとんどのケースで不要なチャンク割り当てとロック取得を回避
- **UnwindInfoTable**: グローバル公開ロック/保留ロックをテーブル単位のロックに変更（`m_publishLock`、`m_pendingLock`を個別管理）
- **公開ロック競合削減**: 全スレッドが公開ロックを取得する必要をなくし、フラッシャースレッドの完了を待つか、フラッシャースレッドに昇格することで競合を軽減
- **パフォーマンス最適化**: 
  - 保留テーブルのサイズ≤4時はSwapソートを使用
  - エントリ削除をバイナリサーチで実装（線形探索から変更）
  - 登録失敗フラグを追加して不要なロック取得を回避

#### パフォーマンスへの影響
**LCGストレスベンチマーク（300K メソッド）:**
- .NET 10: 12.868秒 (.NET 9比+169%)
- .NET 10 + このPR: 7.177秒 (.NET 10比-44%、.NET 9比+50%)

**Original xUnitベンチマーク（30K メソッド）:**
- .NET 10: 2.772秒 (.NET 9比+35%)
- .NET 10 + このPR: 2.181秒 (.NET 10比-21%、.NET 9比+6%)

大規模LCG作成時の競合を大幅に削減しますが、.NET 9比では若干遅化が残存します。
