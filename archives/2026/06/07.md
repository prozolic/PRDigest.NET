#### 概要
`Microsoft.Extensions.Options`に非同期スタートアップ検証機能を追加します。新しい`IAsyncValidateOptions<T>`インターフェースと`IAsyncStartupValidator`により、`Host.StartAsync()`実行時に非同期検証パイプラインが動作します。既存の同期パス（`IOptions<T>.Value`、`OptionsFactory.Create()`）は完全に同期のままであり、既存の動作に影響を与えません。

```csharp
// 非同期検証の例

    .Validate(async (options, ct) => 
    {
        var isValid = await ValidateAsync(options);
        return isValid;
    });
```

#### 変更内容
**System.ComponentModel.Annotations（DataAnnotations）:**
- `AsyncValidationAttribute` — 非同期検証属性の新しい基底クラス
- `IAsyncValidatableObject` — 非同期検証可能オブジェクトの新規インターフェース
- `Validator` — 8個の新しい非同期メソッド追加（`TryValidateObjectAsync`等）

- `IAsyncValidateOptions<T>` — `Task<ValidateOptionsResult>`を返す非同期検証インターフェース
- `IAsyncStartupValidator` — ホストレベルの非同期スタートアップ検証インターフェース
- `AsyncValidateOptions<TOptions>` （0～5依存関係版） — ラムダベースの非同期検証クラス
- `OptionsBuilder<T>`に非同期`Validate`オーバーロード追加（0～5依存関係）
- `StartupValidator` — `IAsyncStartupValidator`も実装（同期検証後に非同期検証を実行）
- `ValidateOnStart()` — 非同期検証エントリ登録に対応

**Microsoft.Extensions.Options.DataAnnotations:**
- `DataAnnotationValidateOptionsAsync<T>` — `Validator.TryValidateObjectAsync`を呼び出し、`[ValidateObjectMembers]`/`[ValidateEnumeratedItems]`を再帰的に処理
- `ValidateDataAnnotationsAsync()` 拡張メソッド追加

**Microsoft.Extensions.Hosting:**
- `Host.StartAsync()` — `IAsyncStartupValidator`を優先して使用、利用不可時は`IStartupValidator`にフォールバック

#### パフォーマンスへの影響
影響なし。非同期検証は`Host.StartAsync()`実行時にのみ別ステップで実行されます。同期パス（`IOptions<T>.Value`、`OptionsFactory.Create()`）は影響を受けません。遅延検証と実行時リロードは非同期検証に影響されません。
