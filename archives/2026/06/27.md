### [#129911](https://github.com/dotnet/runtime/pull/129911) AES-KWP Performance Improvements for Apple {#129911}
- 作成者: [@vcsjones](https://github.com/vcsjones)
- 作成日時: 2026年06月26日 20:24:50(UTC)
- マージ日時: 2026年06月27日 00:41:01(UTC)
- ラベル: <a style="text-decoration:none;" href="../../labels/area-System.Security/index.html"><span style="background-color: #d4c5f9; color: #000000; display: inline-block; padding: 0 7px; font-size:12px; font-weight:500; line-height:18px; border-radius:2em; border:1px solid transparent;">area-System.Security</span></a> <a style="text-decoration:none;" href="../../labels/tenet-performance/index.html"><span style="background-color: #c2e0c6; color: #000000; display: inline-block; padding: 0 7px; font-size:12px; font-weight:500; line-height:18px; border-radius:2em; border:1px solid transparent;">tenet-performance</span></a>
#### 概要
Apple プラットフォーム上での AES-KWP (AES Key Wrap with Padding) パフォーマンスを大幅に改善します。従来は ECB 暗号化/復号化の呼び出し毎にハンドル設定・破棄を繰り返していたのに対し、単一の lite cipher インスタンスを再利用することで 5〜10 倍のパフォーマンス向上と大幅なメモリ割り当ての削減を実現します。

#### 変更内容
- **Aes.cs**: RFC 3394 ラップ/アンラップ処理に ECB 変換デリゲートを注入可能にする内部ヘルパーメソッドを追加。既存の `protected virtual` 実装との互換性を維持。
- **AesImplementation.Apple.cs**: Apple 固有の `EncryptKeyWrapPaddedCore`/`DecryptKeyWrapPaddedCore` オーバーライドを追加。単一の `AppleCCCryptorLite` インスタンスをラップ/アンラップ全体で再利用し、セットアップオーバーヘッドと割り当てを削減。

#### パフォーマンスへの影響
**実行速度の改善:**
- [1, 8] バイト: パフォーマンス変化なし
- (8, 512] バイト: 約 5 倍高速化
- (512 以上) バイト: 約 10 倍高速化

**メモリ割り当ての改善:**
- 入力長 512 バイト以上で劇的に削減。例として 4096 バイト入力の場合、従来は 221,192 バイト割り当てていたのに対し、変更後は 72 バイトに削減（約 3,000 分の 1）。
- 割り当ては入力サイズに関わらず定量化。

**具体例 (ベンチマーク結果より):**
- 16 バイト入力: 3,539.1 ns → 633.2 ns (約 5.6 倍)、864 B → 72 B
- 512 バイト入力: 110,421.3 ns → 11,922.3 ns (約 9.3 倍)、27,649 B → 72 B

#### 関連Issue
なし

#### その他
- Windows プラットフォーム向けの同様の最適化はフォローアップ PR で対応予定。
- 基盤実装の `protected virtual` インターフェースは変更なく、互換性が保たれています。
