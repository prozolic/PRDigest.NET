#### 概要

`System.Runtime.Intrinsics.Arm` API を拡張して `nint`/`nuint` オーバーロードを追加する承認提案を実装します。スコープは `ArmBase` + `Crc32` に限定されます。同時に、JIT の `GenTreeHWIntrinsic` の補助型追跡を `CorInfoType` から `var_types` へリファクタリングし、既存の JIT 型プラッシングとの整合性を向上させます。

#### 変更内容

**マネージド API（公開）**
- `ArmBase`: `LeadingZeroCount(nint)`、`LeadingZeroCount(nuint)`、`ReverseElementBits(nint)`、`ReverseElementBits(nuint)` を追加
- `ArmBase.Arm64`: `LeadingSignCount(nint)` を追加
- `Crc32`: `ComputeCrc32(uint crc, nuint data)`、`ComputeCrc32C(uint crc, nuint data)` を追加
- ref assembly と `PlatformNotSupported` ミラーも更新

**JIT 実装（内部）**
- `hwintrinsiclistarm64.h`: 32 ビット クラスの 64 ビット命令スロット（`TYP_LONG`/`TYP_ULONG`）を入力。`NI_ArmBase_LeadingZeroCount` に `INS_clz`、`NI_ArmBase_ReverseElementBits` に `INS_rbit`、`NI_Crc32_ComputeCrc32` に `INS_crc32x`、`NI_Crc32_ComputeCrc32C` に `INS_crc32cx` を追加
- `GenTreeHWIntrinsic` リファクタリング: 補助型ストレージを `CorInfoType` から `var_types` へ変更（`gtAuxiliaryJitType` → `gtAuxiliaryType`、デフォルト値 `CORINFO_TYPE_UNDEF` → `TYP_UNKNOWN`）
- 約 18 カ所の呼び出しサイトを `hwintrinsic.cpp`、`hwintrinsicarm64.cpp`、`hwintrinsicxarch.cpp`、`gentree.cpp`、`valuenum.cpp` で更新
- 不要になった `getBaseJitTypeAndSizeOfSIMDType`/`getBaseJitTypeOfSIMDType` ヘルパーを削除

**テスト**
- `BaseTests.cs`: `nint`/`nuint` オーバーロードのテストジェネレータ入力を追加。`LeadingZeroCount`/`LeadingSignCount`/`ReverseElementBits` の期待値計算を BCL API で実装し、エッジケース（`data == 0`）でのループ無限化バグを修正。`Crc32` `nuint` オーバーロードは `UIntPtr.Size` 分岐による動的検証

影響なし。マネージド API は単純な intrinsic スタブで、JIT が HWIntrinsic リストで処理を行います。`var_types` リファクタリングは型システムの整理で、既存の型プラッシング機構に統合されており、パフォーマンス低下がありません。
