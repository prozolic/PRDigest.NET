#### 概要
x86のJITコンパイラが浮動小数点型からlong/ulong型へのキャストをハードウェアアクセラレーションする機能を追加しました。AVX-512とAVX10.2を活用することで、オーバーフロー発生時以外のすべてのキャストがハードウェア命令で処理されるようになります。これまではCORINFO_HELP_DBL2LNGなどのヘルパー関数呼び出しが必要でしたが、新しい実装により関数呼び出しなしでインライン化された命令列で処理されます。

サンプルコード（double→long AVX-512）:
```csharp
// 従来: ヘルパー関数呼び出しによる処理
double value = 123.456;
long result = (long)value;  // CORINFO_HELP_DBL2LNG呼び出し

// 新実装: AVX-512命令によるインライン処理
// vmovsd, vcmpordsd, vcmpge_oqsd, vcvttpd2qq, vpblendmqなどで直接処理
```

#### 変更内容
- **decomposelongs.cpp**: AVX-512/AVX10.2を用いた浮動小数点↔long/ulongキャストのローリング/デコンポーズロジックを実装
- **flowgraph.cpp**: x86で浮動小数点↔longキャスト時にAVX-512が利用可能な場合、ヘルパー関数呼び出し回避ロジックを追加
- **hwintrinsiclistxarch.h/hwintrinsicxarch.cpp**: AVX-512スカラー比較マスク命令（`vcmpss/vcmpsd`）を新しいハードウェア組み込み関数として追加
- **gentree.cpp/lowerxarch.cpp**: ベクトル定数構築をリファクタリングし、新しい組み込み関数の制約サポートを追加

#### パフォーマンスへの影響
従来のdouble→longキャストでは31バイトのコード（ヘルパー関数呼び出し、スタック操作含む）が生成されていましたが、新実装では53バイトのコード生成となります。ただし関数呼び出しオーバーヘッド（引数プッシュ、戻り値リターン）がなくなり、CPUキャッシュ効率が向上します。実際のパフォーマンス向上値は具体的に記載されていません。

#### その他
**注記**: Copilotレビューで指摘されているように、現在の実装ではx86のヘルパー判定ロジックがAVX-512のみをチェックしており、AVX10.2が有効でAVX-512が無効の環境でも不要なヘルパー呼び出しが発生する可能性があります。これは別途の修正が必要な潜在的な問題です。
