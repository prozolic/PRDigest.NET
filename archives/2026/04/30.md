- 作成者: [@eiriktsarpalis](https://github.com/eiriktsarpalis)

#### 概要
System.Text.Jsonに4つの新しい公開APIを追加し、`IAsyncEnumerable<TValue>`をJSON Lines (JSONL)形式で直接シリアル化できるようにします。`topLevelValues`パラメータを`true`に設定することで、従来のJSON配列形式（デフォルト）の代わりにJSONL形式（`\n`区切りの行形式）で出力できます。既存の`DeserializeAsyncEnumerable`と同じAPIパターンを採用しています。

```csharp
// 例: JSONL形式でシリアル化
public static Task SerializeAsyncEnumerable<TValue>(
    Stream utf8Json, IAsyncEnumerable<TValue> value,
    bool topLevelValues = false, JsonSerializerOptions? options = null,
    CancellationToken cancellationToken = default);
```

#### 変更内容
- **新しい公開API**: `JsonSerializer.SerializeAsyncEnumerable<TValue>`の4オーバーロード（Stream/PipeWriter、JsonSerializerOptions/JsonTypeInfo<TValue>の組み合わせ）
- **JSONL書き込みロジック**: `JsonSerializer.Write.Stream.cs`と`JsonSerializer.Write.Pipe.cs`に実装。各値の後に正規化されたLF（`\n`）を出力
- **JSONL専用オプション**: `JsonSerializerOptions.GetWriterOptionsForJsonLines()`を追加。Indentedを強制的にfalseとし、Encoderと最大深度は保持
- **キャッシング機構**: `GetOrAddIAsyncEnumerableTypeInfoForSerialize<T>`で`JsonTypeInfo<IAsyncEnumerable<T>>`を合成・キャッシュ。ソースジェン環境でも登録なしで動作
- **PipeWriterサポート**: `FlushResult.IsCompleted`で早期終了、`FlushResult.IsCanceled`で`OperationCanceledException`をスロー
- **ドキュメント更新**: 既存の`DeserializeAsyncEnumerable`と新APIの両方で https://jsonlines.org/ を明示的に引用

#### パフォーマンスへの影響
影響なし。JSONL各行の書き込みごとに`Utf8JsonWriter`を再利用（parameterless `Reset()`経由）し、フラッシュを行うため、ストリーミング向けの標準的なバッファリング動作を踏襲しています。

#### 関連Issue
[#126395](https://github.com/dotnet/runtime/issues/126395)

#### その他
- テストカバレッジ: 462個の新規テスト（JSONL形式での往復テスト、正規の`\n`改行の検証、WriteIndentedの無視、null要素、多態型、バッファ境界ストレステスト、キャンセレーション処理など）
- 部分的なアイテム失敗時の動作: 書き込み済みのアイテムはファイナライズされ、障害アイテムより後の処理は中断（ドキュメント化済み）

- 検証済み: System.Text.Json.Tests および System.Text.Json.SourceGeneration.Roslyn4.4.Tests で各452個・135個のAsyncEnumerableテストが合格
