### [#127434](https://github.com/dotnet/runtime/pull/127434) JIT: Use normalized block for comparison in RBO dominator walk {#127434}
- 作成者: [@hez2010](https://github.com/hez2010)
- 作成日時: 2026年04月26日 14:12:17(UTC)
- マージ日時: 2026年04月26日 18:53:58(UTC)
- ラベル: <span style="background-color: #d4c5f9; color: #000000; display: inline-block; padding: 0 7px; font-size:12px; font-weight:500; line-height:18px; border-radius:2em; border:1px solid transparent; cursor:default;">area-CodeGen-coreclr</span> <span style="background-color: #c2e0c6; color: #000000; display: inline-block; padding: 0 7px; font-size:12px; font-weight:500; line-height:18px; border-radius:2em; border:1px solid transparent; cursor:default;">community-contribution</span>
#### 概要
JIT最適化のRBO（Redundant Branch Optimization）ドミネータウォーク処理において、空の`BBJ_ALWAYS`ブロックチェーンの作成後に早期終了してしまうバグを修正しました。比較対象のブロックを正規化（副作用のないブロックをスキップ）することで、不正なブロック比較を防ぎます。その結果、不必要な条件分岐が削除され、コード生成が改善されています。

#### 変更内容
- **src/coreclr/jit/redundantbranchopts.cpp**: ドミネータウォーク処理でブロック比較前に`skipSideEffectFreeBlocks`でブロックを正規化し、空の`BBJ_ALWAYS`チェーンによる比較ミスマッチを防止
- **src/tests/JIT/opt/RedundantBranch/RedundantBranchDominating.cs**: ネストされたドミネーティング分岐シナリオ（`Dom_05`）の新規テストケース追加

#### パフォーマンスへの影響
コード生成が最適化され、冗長な条件分岐が削除されました。具体的には：
- 生成コードサイズが21バイトから16バイトに削減（約24%削減）
- PerfScoreが2.88から2.25に改善（約22%改善）
- 不必要な条件分岐チェックが除去される

#### 関連Issue
[#127432](https://github.com/dotnet/runtime/issues/127432)

#### その他
本修正はJITコンパイラの内部最適化パス（`optRedundantDominatingBranch`）に対する修正であり、公開APIや言語機能には影響を与えません。既存の最適化結果の正確性向上が目的です。

### [#127433](https://github.com/dotnet/runtime/pull/127433) JIT: relax inliner heuristics for callees on \[Intrinsic\] types {#127433}
- 作成者: [@EgorBo](https://github.com/EgorBo)
- 作成日時: 2026年04月26日 12:26:16(UTC)
- マージ日時: 2026年04月26日 17:02:08(UTC)
- ラベル: <span style="background-color: #d4c5f9; color: #000000; display: inline-block; padding: 0 7px; font-size:12px; font-weight:500; line-height:18px; border-radius:2em; border:1px solid transparent; cursor:default;">area-CodeGen-coreclr</span>
#### 概要
"unsafe削除"の取り組みで導入される`Span.Slice`や`Vector.Create`など、`[Intrinsic]`属性が付与されたメソッドの呼び出しが、JITインライナーの時間予算に負の影響を与えてコード品質が低下している問題を解決します。本PRは、これらのintrinsic型のメソッドをインラインする際の予算チェックを緩和し、小規模メソッドと同様の扱いをすることで、回帰を防ぎます。

#### 変更内容
- **inlinepolicy.h/cpp**: 新しい観察値`CALLEE_IS_INTRINSIC_TYPE`を追加し、intrinsic型呼び出しに対する予算制限と乗数ペナルティ（レアコールサイト、プロファイルベースのペナルティ、非リターン領域キャップ）を緩和
- **inline.def**: callee scopeで`IS_INTRINSIC_TYPE`観察値を定義
- **fgbasic.cpp**: インライン候補スキャン時にコンパイル情報から`CALLEE_IS_INTRINSIC_TYPE`を記録
- **inline.cpp**: intrinsic型のstate管理を追加

#### パフォーマンスへの影響
コードジェン品質の改善が期待されます。特にSpan関連やSIMDベクトル、ハードウェアintrinsicを多用するコードでの回帰防止が目的です。PMI jit-diffsで数件のヒット報告があります。

#### 関連Issue
[#127429](https://github.com/dotnet/runtime/pull/127429/)

#### その他
- 内部実装（JITコンパイラ）の変更であり、公開APIへの影響なし
- 互換性に関する破壊的変更なし

- 作成者: [@maraf](https://github.com/maraf)

- マージ日時: 2026年04月26日 16:28:59(UTC)
- ラベル: <span style="background-color: #eb6420; color: #ffffff; display: inline-block; padding: 0 7px; font-size:12px; font-weight:500; line-height:18px; border-radius:2em; border:1px solid transparent; cursor:default;">arch-wasm</span> <span style="background-color: #d4c5f9; color: #000000; display: inline-block; padding: 0 7px; font-size:12px; font-weight:500; line-height:18px; border-radius:2em; border:1px solid transparent; cursor:default;">area-System.Diagnostics</span> <span style="background-color: #f49cb1; color: #000000; display: inline-block; padding: 0 7px; font-size:12px; font-weight:500; line-height:18px; border-radius:2em; border:1px solid transparent; cursor:default;">test-enhancement</span> <span style="background-color: #FEF2C0; color: #000000; display: inline-block; padding: 0 7px; font-size:12px; font-weight:500; line-height:18px; border-radius:2em; border:1px solid transparent; cursor:default;">os-browser</span>
#### 概要
ブラウザ(WebAssembly)をターゲットする場合に`System.Diagnostics.StackTrace.Tests`で`StackTraceLineNumberSupport`を有効化し、スタックトレースのファイル/行番号情報をサポートします。これにより、CoreCLR+browserでPDB情報に依存するテストが`[ActiveIssue]`でスキップされずに実行できるようになります。

#### 変更内容
- **System.Diagnostics.StackTrace.Tests.csproj**: ブラウザターゲット時に`StackTraceLineNumberSupport=true`を設定（外部からのオーバーライドを許可）
- **StackTraceTests.cs**: `[ActiveIssue(#50957)]`の条件から`PlatformDetection.IsBrowser`を削除し、`IsMonoAOT`のみに制限（7行変更）
- **StackFrameTests.cs**: 同様にスキップ条件を`IsMonoAOT`に絞り込み（4行変更）
- **StackFrameExtensionsTests.cs**: 同様にスキップ条件を`IsMonoAOT`に絞り込み（2行変更）

#### パフォーマンスへの影響
影響なし

#### 関連Issue
[#50957](https://github.com/dotnet/runtime/issues/50957)、[#127147](https://github.com/dotnet/runtime/pull/127147)の review feedbackに対応

#### その他
WebAssembly SDKはデフォルトで`DebugSymbols`が有効でない場合に`StackTraceLineNumberSupport`を`false`に設定し、スタックトレースから行番号情報を削除します。本変更により、既に設定されている`DebuggerSupport=true`とPDBファイル包含との組み合わせで、ブラウザ環境でのスタックトレース機能を完全に活用できるようになります。
