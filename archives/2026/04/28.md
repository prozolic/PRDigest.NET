#### 概要
JITコンパイラの最適化を強化し、`span.Slice(span.Length - cns)` パターンで発生する冗長な境界チェックを削除します。値番号付け(VN)の恒等式拡張と範囲チェック(RangeCheck)のアサーション推論改善により、`BinaryPrimitives.ReadInt32BigEndian` 内の長さチェックなども折り畳み可能になります。

生成されるアセンブリコードが大幅に削減され、例示コードでは stackalloc や冗長な境界チェック呼び出しが完全に除去されています。

#### 変更内容
- **src/coreclr/jit/valuenum.cpp**: 減算の新しい恒等式を追加し、`x - (x + a)` / `x - (a + x)` を `-a` に値番号付けします（非浮動小数点、非オーバーフロー時）。これにより `length - (length - cns)` パターンが定数に簡約され、後続チェックの折り畳みが可能になります。

- **src/coreclr/jit/rangecheck.cpp**: 関連する関係演算子(relop)の折り畳みロジックを追加し、負の定数 `K < 0` について `ADD(A, K) < A` を証明できるようにします。`A` の下限が `-K` 以上の場合、ラップアラウンドが発生しないと判断し、`Slice(start)` の `(uint)start > (uint)_length` チェックを削除します。

#### パフォーマンスへの影響
改善が顕著です。提供されたコード例では：
- stack領域の確保・解放が不要になり (`sub/add rsp` が削除)
- 冗長な境界チェック処理が完全に除去
- 例外スロー呼び出し(`ThrowArgumentOutOfRangeException`)が不要化
- 生成コード量が大幅削減（複数の分岐やチェックロジックが消去）

**懸念点**: なし。純粋な冗長性除去のため、正確性に影響はありません。

#### 概要

```csharp
internal static void FinishSuspensionNoContinuationContext(
    ref Continuation continuation,
    bool hasExecutionContext,
    ExecutionContext currentContext,

    ref Continuation continuation,
    bool hasExecutionContext,
    ExecutionContext currentContext,
    SynchronizationContext syncContext);
```
